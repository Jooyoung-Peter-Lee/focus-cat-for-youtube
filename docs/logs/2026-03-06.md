## 🔧 Technical Notes

### Warning Banner (`warningBanner.ts`)

- `BANNER_ID = 'focus-cat-warning-banner'`
- `showWarningBanner(player, { remainingSeconds })` — idempotent
- `hideWarningBanner()` — immediate removal, cancels auto-dismiss timer

- Warm amber gradient — encouraging tone, not alarming
- `animateOutAndRemove()` uses `transitionend` + `setTimeout` fallback + `cleaned` guard

### Race Condition Fix

**Root cause:** `player.appendChild(banner)` ran synchronously mid-frame, colliding
with YouTube's Polymer renderer mid-`insertBefore`, throwing:
`TypeError: Failed to execute 'insertBefore' on 'Node': parameter 1 is not of type 'Node'`

**Fix applied:**
1. `player.isConnected` guard before building the element
2. `requestAnimationFrame` deferral — append runs after YouTube's current DOM batch settles
3. Both idempotency and `isConnected` re-checked inside the rAF callback
4. `try/catch` around `player.appendChild` — logs warning, returns cleanly on failure
5. `autoDismissTimer` only set after a successful mount

### Overlay Controller Changes

Added two new state variables:
- `desiredBanner` — parallel to `desiredShown`; stays `true` while state is `'warning'`
- `lastWarningState` — stores the `'warning'` LimitState for retries

`updateOverlayState('warning')` now defers to `requestReattach()` instead of mounting
inline — same reattach mechanism used by the full overlay, SPA-safe.

`doReattach()` has two explicit paths: banner path (checked first) and overlay path.

`requestReattach()` guard updated: `!desiredShown && !desiredBanner` so banner
participates in the debounced retry loop driven by `playerWatcher` / `navigationWatcher`.
